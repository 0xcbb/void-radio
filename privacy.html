隐私政策

最后更新：2026 年 7 月 13 日

Void Radio 是一个零账号、零后端优先的网络收音机。我们尽量让播放、收藏、精选电台库、地图发现、私人微电台和分享都在你的设备本地完成。

我们不收集的内容

  • 不要求注册账号。
  • 不运营 Void Radio 自有用户数据库。
  • 不收集通讯录、照片、麦克风录音或本地文件。
  • 不把你的收藏列表、喜欢列表、坏源标记上传到 Void Radio 服务器。

位置权限

当你使用“附近电台”或“地球背面电台”时，App 可能请求系统定位权限，用来在设备本地计算附近坐标、对跖点坐标，并向公共电台目录查询匹配电台。你可以在系统设置里随时关闭定位权限。

第三方数据与直播流

Void Radio 使用 Radio-Browser 等公开电台目录和电台公开直播流。查询国家、城市、附近电台或播放直播流时，第三方服务可能会收到必要的请求信息，例如 IP 地址、查询参数、设备网络信息或浏览器/播放器请求头。这些服务不由 Void Radio 控制。

电台目录、地图瓦片、反向地理编码和直播流由第三方服务或原始广播方提供。Void Radio 会优先使用 HTTPS API；但部分公开直播流可能仍由原始广播方以 HTTP 提供，传输协议取决于该电台源站。

曲目信息与实验性歌词

当电台直播流没有返回曲目信息时，Void Radio 可能查询广播方使用的 Triton Digital Now Playing 服务。启用“实验性歌词”后，App 会把当前曲目名称和艺人名称发送给 LRCLIB 查询歌词；直接查询未命中时，可能向 MusicBrainz 查询规范曲名、发行信息和艺人别名，再重试 LRCLIB。Void Radio 不运营这些服务，也不会把歌词查询用于广告或用户画像。你可以在设置中关闭实验性歌词。

声音明信片与播放链接

当你主动分享播放链接时，链接中可能包含电台名称、地区、频率、坐标、当前曲目文字、直播流地址和备用流地址。收到链接的人可以在浏览器里打开并播放。请只分享你愿意公开的信息。

私人微电台与个人存储

当你主动添加群晖、QNAP 或其他 NAS SMB、WebDAV、AList WebDAV 连接时，App 会直接从你的设备连接到你填写的存储地址。Void Radio 不运营中转服务器，不会接收或保存你的 NAS 账号、密码、Token、目录结构、文件名或音频内容。

连接名称、地址、共享目录、账号、域和安全选项保存在设备本地；密码或 Token 使用 Android Keystore 等系统安全能力加密保存。SMB 音频通过仅监听设备本机 127.0.0.1 的临时随机令牌通道交给播放器，不会把密码写入播放地址、媒体信息或分享链接。删除连接时，App 会同时删除对应的本地加密凭据。

SMB 连接仅面向局域网、本地主机或用户自行建立的 Tailscale 等私有网络。WebDAV / AList 的网络运营方、NAS 厂商和用户选择的网络服务可能按照各自政策处理必要的 IP 地址和连接日志，这些服务不由 Void Radio 控制。

本地存储

App 会在设备本地保存部分偏好设置，例如首页电台库、排序、收藏、喜欢、主题、显示语言、触感反馈、歌词开关、数据节省模式、坏源标记和私人存储的非秘密连接配置。私人存储密码或 Token 使用系统安全存储单独加密。卸载 App 通常会删除这些本地数据。

联系我们

隐私问题、坏源反馈或删除请求可以发送到 support@feedar.cc。

这份政策会随 Void Radio 的功能变化更新。若后续加入录音、账号、云同步或 Void Radio 自有后端，会在发布前补充对应隐私说明。
